# Portfolio Project Rules

## Changelog 規則

每次修改任何專案內的檔案後，**必須更新 `CHANGELOG.md`**，格式如下：

1. 在最上方新增日期區塊（若當日已有則直接追加）
2. 每筆記錄包含：
   - 異動檔案名稱與行號
   - 完整 diff（`-` 原始碼 / `+` 修改後程式碼）
   - 修改原因（技術說明或設計決策）

### 範例格式

```
## YYYY-MM-DD

### [N] 檔案名稱 — 修改摘要

**filename.js — Line XX**
```diff
- 原始程式碼
+ 修改後程式碼
```

> 原因：說明為什麼這樣改。
```

## 其他規則

- 回應語言：台灣繁體中文
- 修改前先確認行號，確保 diff 精確可還原
